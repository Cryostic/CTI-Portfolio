<h2>ATT&CK Kill Chain Mapping (Unified Kill Chain: IN --> THROUGHT --> OUT)</h2>

<h3>IN: Initial Access & Execution</h3>

| Technique | ATT&CK ID | Description & Context |
| --- | --- | --- |
| Exploit-Public Facing Application | T1190 | **PRIMARY VECTOR**; exploitation of VPN devices, Exchange servers, Atlassian Confluence, Log4j, WinRAR; preference confirmed by eight-nation advisory |

| External Remote Services (VPN) | T1133 | Exploitation of VPN credential stores; compromised VPN credentials as persistent access mechanism |

<h4>APT40's Preferred Exploitation Sequence</h4>
```
RECONNAISSANCE (ScanBox, MURKYTOP, open-source tools)
└── Target identification: organizations running vulnerable public-facing software
    └── CVE disclosed + PoC released (hours to days):
        └── APT40 adapts PoC → weaponized exploit developed
            └── PUBLIC-FACING SERVER exploited (VPN, Exchange, Confluence, etc.)
                └── Web shell deployed → persistent server-side access established
                    └── CREDENTIAL HARVESTING begins (HOMEFRY, Windows Credential Editor)
                        └── Valid credentials obtained → malware dependency REDUCED
                            └── Lateral movement via legitimate credentials (RDP, SSH, PsExec)
                                └── KEYPLUG / BADFLICK / BEACON deployed as needed
                                    └── Long-term persistence and data collection
```
